# Allbert v0.14 Session Scratchpad And Active App Context

Formerly tracked as M-D1b.

## Purpose

v0.14 adds volatile session state keyed by `{user_id, session_id}`. The
scratchpad lets Allbert remember active app context and short-lived working
state during a local session without treating ETS as durable memory or a
security boundary.

## Expected Inputs

- v0.12 local workspace identity and conversation history exist.
- ADR 0014 is accepted.
- The runtime accepts `session_id` and propagates `user_id`.

## Expected Direction

- Add `AllbertAssist.Session.Scratchpad` as an ETS-backed process started in
  the supervision tree.
- Key entries by `{user_id, session_id}`.
- Store `%{active_app: atom | nil, working_memory: map, expires_at: integer}`.
- Default TTL is 30 minutes and later settings-configurable.
- Runtime reads scratchpad when `session_id` is present and adds `active_app`
  to request/decision context.
- `active_app` defaults to nil for general Allbert context.
- Reserve `canvas_tiles` for v0.26; do not define or populate it here.

## Non-Goals

- No durable session persistence.
- No auth/session-token model.
- No security isolation claim from ETS.
- No canvas state.
- No app registry; that is v0.15.

## Test Plan

- Entries for `{alice, sess-1}` and `{alice, sess-2}` are isolated.
- Entries for Alice and Bob with the same session id are isolated.
- Expired entries return nil and are swept.
- Application restart starts with an empty scratchpad.
- Runtime includes `active_app` only when a scratchpad entry exists.

## Definition Of Done

v0.14 is done when session scratchpad state is supervised, TTL-expiring,
runtime-visible, and covered by focused tests without changing durable memory
or authorization behavior.
