# Allbert v0.09 Execution-Aware Intent Contract

## Purpose

v0.09 introduces the first structured intent decision contract over real risky
capabilities.

The original v0.09 plan tried to enrich intent across jobs, channels, memory,
and model-assisted routing. That remains important, but Allbert now needs a
nearer milestone that makes shell, script, package, network, and online import
decisions explainable before scheduled jobs or new channels can consume those
capabilities.

## Expected Inputs

- v0.04 action-backed skills and action runner exist.
- v0.05 confirmation workflow exists.
- v0.06 shell execution exists behind sandbox policy.
- v0.07 skill script execution exists behind trusted resource policy.
- v0.08 external service, package-install, and online import adapters exist.
- Permission decisions remain centralized in
  `AllbertAssist.Security.PermissionGate`.

## Non-Goals

- No new execution powers.
- No model-selected side effects without validation.
- No broad cross-channel routing; v0.13 owns that.
- No memory retrieval or embeddings work.
- No channel-specific intent fork.

## Intent Decision Contract

Introduce a structured intent decision, tentatively
`AllbertAssist.Intent.Decision`, with fields equivalent to:

- `:intent` such as `:direct_answer`, `:activate_skill`,
  `:run_skill_action`, `:run_shell_command`, `:run_skill_script`,
  `:install_package`, `:external_service_request`, `:import_skill`,
  `:needs_confirmation`, `:unsupported`, or `:deny`
- `:confidence`
- `:reason`
- `:risk_summary`
- `:selected_skill`
- `:candidate_skills`
- `:selected_action`
- `:candidate_actions`
- `:permission`
- `:confirmation`
- `:execution_mode`
- `:foreground_or_background`
- `:alternatives`
- `:diagnostics`
- `:trace_metadata`

The decision object is plain data. It does not execute anything. The runtime
still executes through registered actions, the action lifecycle runner, the
permission gate, and confirmation state.

## Expected Direction

- Add the decision struct or plain map schema and render it in traces.
- Move current route predicates behind a decision-producing module while
  preserving existing behavior.
- Validate every selected skill/action/permission/execution mode against the
  registry, action runner, permission gate, confirmation state, and Settings
  Central policy.
- Add deterministic decision coverage for shell command, skill script, package
  install, external service, online skill import, refusal, and safe fallback
  cases.
- Keep optional model assistance out until v0.13 unless it is fully fixture-
  injected and validates against known names.

## Definition Of Ready

v0.09 is ready to implement when:

- v0.06 through v0.08 expose risky capabilities as registered actions.
- Action results and traces can carry selected skill/action/permission context.
- Confirmation state can distinguish pending, approved, denied, expired, and
  not-required decisions.

## Definition Of Done

v0.09 is done when:

- Runtime turns can include an intent decision object.
- Existing safe v0.08 behaviors are preserved.
- Risky capability prompts produce inspectable selected/candidate decisions
  before execution or confirmation.
- Invalid model or route output cannot invent skill names, action names,
  permissions, or execution modes.
- Intent traces explain selected and rejected candidates.
