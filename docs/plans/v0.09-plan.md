# Allbert v0.09 Skill Script Runner

## Purpose

v0.09 lets trusted Agent Skills run selected bundled scripts, but only through
Allbert's action, confirmation, sandbox, Security Central, and trace
boundaries.

v0.03 already inventories `scripts/` as inert resources. v0.08 provides a
Level 1 local policy sandbox, not OS/container isolation. v0.09 connects those
pieces with a stricter rule: scripts are executable only when they are part of
the selected skill's parsed resource inventory and a registered action chooses
to run them.

## Expected Inputs

- v0.03 resource inventory includes bundled skill scripts.
- v0.05 Security Central can evaluate trust boundaries and redaction policy.
- v0.06 validates trusted skill capability contracts against registered
  actions.
- v0.07 confirmation workflow can approve or deny risky actions.
- v0.08 Level 1 local execution sandbox can execute bounded commands with
  explicit confirmation, trace metadata, and execution audit records.
- ADR 0009 defines sandbox levels and defers container, remote, and microVM
  isolation to future adapter work.

## Expected Direction

- Add `run_skill_script` as a registered Jido action with selected skill, script
  resource path, args, working-directory policy, timeout, and expected output
  mode.
- Resolve script paths only from the selected skill's parsed resource
  inventory; reject absolute paths, path traversal, hidden files, missing
  resources, and non-script resources.
- Require the skill to be trusted, enabled, selected for the current action
  context, and allowed by its validated capability contract.
- Run scripts through the v0.08 sandbox and v0.07 confirmation workflow.
- Treat the v0.08 sandbox as host execution with policy controls. Add
  script-specific provenance, resource digest, allowed-root, env, timeout, and
  output checks rather than relying on the sandbox level alone.
- Reassess whether any script profiles need Level 2 trusted project/process
  sandboxing before enabling broader script classes.
- Trace script source path, resource digest, args summary, permission,
  confirmation, stdout/stderr summary, exit status, timeout, and failure reason.

## Non-Goals

- No arbitrary script execution outside trusted skill resource inventory.
- No runtime loading of Elixir modules from skill folders.
- No package installs or dependency bootstrap; v0.10 owns install policy.
- No external service adapter from script metadata.
- No persistent background script processes.
- No auto-enabling imported or untrusted skills.
- No Docker, Podman, Mac/Linux container, remote, or microVM backend unless a
  later plan graduates that future work.

## Definition Of Done

v0.09 is done when:

- `run_skill_script` exists only as a registered action.
- Trusted inventoried scripts can run after confirmation through the sandbox.
- Untrusted, disabled, non-selected, non-inventoried, or path-escaping scripts
  are refused before execution.
- Traces include skill, script, resource digest, security decision, and
  execution outcome.
- Skill activation alone still never executes scripts.
