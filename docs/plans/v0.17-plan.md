# Allbert v0.17 StockSage Umbrella App And Domain

Formerly tracked as M-D2a.

## Purpose

v0.17 introduces StockSage as the first domain app inside the Allbert
workspace. It adds the `stocksage` umbrella app, SQLite-first domain storage,
local skill pack, and import path from the existing Python `stocksage.db`.

## Expected Inputs

- v0.12 local string `user_id` exists.
- v0.15 minimal app registration contract exists.
- `aiworkspace-plan.md` has been folded into `allbert-jido-vision.md` as
  durable product direction.

## Expected Direction

- Add `apps/stocksage/` as the OTP core app.
- Add `apps/stocksage_web/` as a stub Phoenix surface app for later v0.22
  LiveViews.
- Implement `StockSage.App` using the v0.15 lite app contract.
- Add SQLite-backed domain records for `Analysis`, `AnalysisDetail`,
  `Outcome`, `AnalysisQueue`, `QueueRun`, and `MemoryEntry`.
- Every user-facing/request/queue row carries string `user_id`.
- Add StockSage local skill pack: `run-analysis`, `get-trends`, and
  `queue-analysis`.
- Add `mix stocksage.import_sqlite /path/to/stocksage.db`.
- Preserve the no-server local posture: no PostgreSQL, no Oban hard
  dependency.

## Non-Goals

- No Python bridge execution; that is v0.19.
- No native Jido trading agents; that is v0.21.
- No StockSage LiveViews; that is v0.22.
- No account foreign keys.

## Test Plan

- StockSage domain records round-trip through SQLite.
- Python `stocksage.db` import is idempotent for representative fixtures.
- StockSage skills appear in `mix allbert.skills list`.
- No local PostgreSQL server is required.

## Definition Of Done

v0.17 is done when StockSage exists as a registered local Allbert app with
SQLite-backed domain records, importable historical data, and discoverable
skills.
