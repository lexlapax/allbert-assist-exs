# Allbert v0.26 Agentic Workspace Surface And Ephemeral UI Substrate

Formerly v0.27, then v0.24 (before the project-direction rethink moved
it after v0.25 native agents so workspace renders real native-agent
output), then v0.26.

## Status

Status: implemented through M20 closeout on 2026-05-18 and ready for
operator manual validation. Version metadata is `0.26.0`; release tag
pending operator acceptance.

This milestone is the **foundational first stage of the Allbert UI
layer**. It upgrades `AllbertAssist.App.CoreApp`'s declared `/agent`
surface from a single prompt+response page into a fully-dynamic
workspace shell — a Surface tree composed of regions, tiles, and
ephemeral surfaces — backed by per-thread persistent canvas state,
strict + HMAC-signed runtime fragment emission, full WCAG 2.1 AA
accessibility, dark mode + mobile responsive layout, and browser-side
offline text editing.

This pass bakes in the decisions taken during the fifth-pass
validation: per-thread Canvas + per-thread Ephemeral Surface
vocabulary, fully-dynamic Surface-tree workspace shell, hybrid
SQLite-metadata + YAML-body canvas persistence, multi-tab sync via
PubSub, full 42-component catalog (including reserved StockSage
cards), `allbert.workspace.fragment.**` signal-driven emission with
strict + HMAC-signed envelope validation, internal AG-UI bridge,
browser-side Yjs + IndexedDB offline text editing, and the v0.07
plan/request-flow split (plan owns acceptance + per-M smokes; flow
stays purely runtime/user flow).

## Implementation Research Notes

Docs consulted during the 2026-05-18 readiness pass:

- Phoenix LiveView docs: use declarative JS commands for simple UI
  transitions where possible; use hooks for client-owned DOM and
  browser APIs. The offline editor is the right place for a hook
  (`mounted`/`updated`/`destroyed`, `pushEvent`, and scoped
  `phx-update="ignore"`); the workspace shell itself should remain
  server-rendered LiveView.
- Yjs docs: the natural browser contract is `Y.Doc`, shared text/map
  types, state vectors/update payloads, and `y-indexeddb`
  persistence. v0.26 therefore uses browser-side Yjs and stores
  opaque bounded update blobs/snapshots server-side instead of
  adding a Rust NIF.
- Tailwind docs: dark mode, contrast variants, motion-reduction
  variants, and explicit focus outlines are all available without
  custom frontend framework choices.

## Purpose

v0.26 turns the Allbert `/agent` LiveView into a real workspace. It
is the first user-facing milestone of the UI layer — operators will
spend most of their time looking at v0.26's output. Every subsequent
UI-touching milestone (v0.27 StockSage LiveViews, v0.29 StockSage
polish, v0.30 canvas integration, post-v0.31 protocol interop) builds
on what v0.26 ships.

The milestone delivers:

1. **Workspace shell as a Surface tree** — the workspace itself is
   declarative, validated against the catalog, and rendered by a
   uniform dispatch from Surface nodes to LiveComponents.
2. **Per-thread Canvas** — persistent tiles bound to v0.12 threads;
   survive browser refresh, LiveView reconnect, app restart.
3. **Per-thread Ephemeral Surfaces** — task-scoped overlays bound
   to thread lifecycle; shared across tabs of the same thread;
   GC'd when the thread closes.
4. **Runtime Fragment emission contract** — any in-BEAM module can
   emit a Tile or Ephemeral Surface by publishing a HMAC-signed
   `FragmentEnvelope` to `allbert.workspace.fragment.**` SignalBus
   topic; receiver validates strictly.
5. **42-component catalog** — v0.18's 12 + 30 v0.26 additions
   (workspace structural, Allbert-domain, app cards, reserved
   StockSage cards).
6. **WCAG 2.1 AA accessibility** — keyboard nav, ARIA, screen
   readers, focus traps, skip-to-content.
7. **Dark mode + theme** — operator-configurable; Tailwind dark
   variants.
8. **Mobile responsive** — CSS-only adaptive layout; two-pane
   above 768px, single-pane below.
9. **Offline text/markdown tile editing** — browser-side Yjs
   documents persisted to IndexedDB with bounded reconnect sync and
   conflict-banner UX; the server stores opaque update blobs and
   human-readable snapshots without a Rust NIF.
10. **Internal AG-UI bridge** — translates a curated subset of
    Allbert SignalBus events to AG-UI event shape (test-only, no
    HTTP exposure; future-proofing for post-v0.31).

It is NOT:

- A new app contract (ADR 0015 is the contract; v0.26 amends only
  the catalog allow-list).
- A new plugin extension point (v0.18 SurfaceProvider stays
  unchanged).
- A new permission framework (Security Central authority unchanged).
- A free-floating shell — it IS `CoreApp`'s implementation of its
  declared `/agent` surface.
- A StockSage-specific surface (the 4 reserved StockSage cards exist
  only as catalog entries with stub rendering modules; real
  rendering ships in v0.27).

## Architecture Readiness Audit

- **ADR 0006 (Security Central)**: every effectful action triggered
  from the workspace flows through `Actions.Runner.run/3`, Security
  Central, confirmations (when needed), audit. The workspace renders
  state and dispatches action invocations through registered actions
  only. Fragments are advisory (per ADR 0021 §4); cannot grant
  permission, bypass confirmation, or invent unregistered actions.
- **ADR 0007 (Jido-Native Internal Runtime Boundaries)**: the
  workspace shell LiveView is a Phoenix LiveView (web-side process),
  not a Jido.Agent. Per the v0.23 pragmatic substrate rule, no new
  JidoBacked agent is required — the canvas state machine is simple
  enough (CRUD + bounded browser-update reconciliation) for plain modules + Ecto. v0.26 does NOT
  introduce a `Workspace.Canvas.Agent` JidoBacked module. If future
  needs (multi-canvas, snapshot/undo, plugin-region contribution)
  emerge, v0.27+ may revisit.
- **ADR 0008 (Durable Confirmation Requests)**: workspace ephemeral
  surfaces include confirmation cards (v0.07/v0.24 confirmations
  rendered through `:confirmation_card` catalog component). Same
  durable storage, same approval flow — workspace just renders
  differently.
- **ADR 0014 (Local Workspace Identity)**: all canvas + ephemeral
  state filters by `user_id`. Cross-user access returns `:not_found`.
- **ADR 0015 (Allbert App Contract And Surface DSL)**: v0.26 amends
  the catalog (12 → 42) but does NOT change the SurfaceProvider
  behaviour, the Surface struct shape, or the validation contract.
  Apps that declared surfaces against the v0.18 catalog continue to
  work.
- **ADR 0017 (Allbert Plugin Contract)**: plugins do not contribute
  workspace regions in v0.26 (post-v0.31 work). Plugins MAY
  contribute Surface declarations targeting catalog components
  (existing v0.18 path), and plugins emit Fragments via the
  SignalBus topic (existing v0.26 emission path).
- **ADR 0019 (Cross-Surface Intent Enrichment)**: workspace renders
  intent candidates in the chat region's trace inspector. Intent
  candidates stay proposal-only; selection still routes through
  Security Central.
- **ADR 0021 (Intent, Objective, Capability, And Advisory Boundary)**:
  the workspace renders objective state + step lists + acceptance
  evaluator output. Advisory rendering only; the engine remains
  authoritative.
- **ADR 0022 (Native Financial Specialist Agents)**: v0.26 reserves 4
  StockSage analysis cards (`:analysis_card`, `:agent_report_card`,
  `:parity_card`, `:debate_round_card`) in the catalog. v0.27
  implements the LiveComponent rendering modules; v0.26 ships
  placeholder stubs.
- **ADR 0023 (NEW — Workspace Canvas And Ephemeral Surface
  Substrate)**: binds vocabulary, persistence model, ephemeral
  scope, Fragment emission/validation, catalog expansion, offline
  editing, AG-UI bridge, UX qualities, permission class, settings
  keys, signal topics, trace integration.

## Guiding Rules

1. **Workspace shell IS a Surface tree.** No hardcoded HEEx layout
   for regions. Renderer dispatches Surface node `:component` atom
   to LiveComponent module. This is the foundational decision in
   ADR 0023 §2.

2. **Authority unchanged.** Every workspace-triggered effect flows
   through `Actions.Runner.run/3` + Security Central + confirmations.
   Fragments are advisory only (ADR 0021 §4). Workspace adds zero
   new authority paths.

3. **Strict validation on every Fragment.** Per ADR 0023 §5:
   FragmentEnvelope shape + HMAC signature + Surface catalog
   validation + emitter allow-list + per-emitter rate limit + size
   cap. Invalid fragments dropped with bounded log; never rendered.

4. **Canvas persists per-thread.** Per ADR 0023 §3+§4: SQLite
   metadata + YAML body. Survives refresh, reconnect, restart. GC'd
   when thread closes.

5. **Ephemeral surfaces per-thread, shared across tabs.** Per ADR
   0023 §4. Multi-tab broadcast via PubSub.

6. **All UX qualities first-class.** Dark mode + WCAG 2.1 AA +
   mobile responsive + offline text/markdown tile editing are v0.26
   deliverables, not v0.27 polish. Per ADR 0023 §10 + §7.

7. **v0.07 plan/request-flow split.** Plan owns implementation +
   acceptance + per-milestone smokes. Flow stays purely runtime
   request-flow + user flow. No `@spec` or settings schema in flow
   doc.

8. **No drag-drop in v0.26.** Resize-bar between regions only. Tile
   rearrange deferred to v0.27+ when StockSage LiveViews drive the
   need.

9. **AG-UI bridge is internal-only.** No HTTP exposure in v0.26.
   Future-proofing only.

10. **StockSage card stubs.** The 4 StockSage analysis cards exist
    as catalog entries with placeholder LiveComponents in v0.26.
    v0.27 ships the real rendering. v0.26 does NOT pre-bake StockSage
    domain rendering logic.

11. **Foundation must run without StockSage.** StockSage is the
    richest proving emitter, not a foundation dependency. The
    workspace shell, catalog, fragment validation, canvas,
    ephemerals, offline editing, and core cards must all pass with
    the StockSage plugin disabled. StockSage-specific cards remain
    reserved stubs and never leak domain rules into the core
    workspace modules.

### `AllbertAssist.Workspace` (facade)

File: `apps/allbert_assist/lib/allbert_assist/workspace.ex`

```elixir
defmodule AllbertAssist.Workspace do
  @moduledoc """
  Public facade for the workspace canvas + ephemeral surface
  substrate. Web-side LiveViews call into this; never reach into
  Canvas / Ephemeral / Fragment modules directly.
  """

  alias AllbertAssist.Workspace.Canvas
  alias AllbertAssist.Workspace.Ephemeral
  alias AllbertAssist.Workspace.Fragment

  @spec canvas_tiles(thread_id :: String.t(), user_id :: String.t()) ::
          {:ok, [Canvas.tile()]} | {:error, term()}
  defdelegate canvas_tiles(thread_id, user_id), to: Canvas, as: :tiles_for_thread

  @spec add_tile(attrs :: map()) :: {:ok, Canvas.tile()} | {:error, term()}
  defdelegate add_tile(attrs), to: Canvas

  @spec update_tile(tile_id :: String.t(), attrs :: map()) ::
          {:ok, Canvas.tile()} | {:error, term()}
  defdelegate update_tile(tile_id, attrs), to: Canvas

  @spec remove_tile(tile_id :: String.t(), user_id :: String.t()) ::
          :ok | {:error, term()}
  defdelegate remove_tile(tile_id, user_id), to: Canvas

  @spec pin_tile(tile_id :: String.t(), user_id :: String.t()) ::
          {:ok, Canvas.tile()} | {:error, term()}
  defdelegate pin_tile(tile_id, user_id), to: Canvas

  @spec unpin_tile(tile_id :: String.t(), user_id :: String.t()) ::
          {:ok, Canvas.tile()} | {:error, term()}
  defdelegate unpin_tile(tile_id, user_id), to: Canvas

  @spec restore_tile(tile_id :: String.t(), user_id :: String.t()) ::
          {:ok, Canvas.tile()} | {:error, term()}
  defdelegate restore_tile(tile_id, user_id), to: Canvas

  @spec ephemeral_surfaces(thread_id :: String.t(), user_id :: String.t()) ::
          {:ok, [Ephemeral.surface()]} | {:error, term()}
  defdelegate ephemeral_surfaces(thread_id, user_id), to: Ephemeral, as: :surfaces_for_thread

  @spec emit_fragment(envelope :: Fragment.envelope()) ::
          :ok | {:error, term()}
```

### `AllbertAssist.Workspace.Canvas` / `Workspace.Ephemeral` / `Workspace.Fragment`

Pure Elixir modules. Canvas + Ephemeral use `AllbertAssist.Repo` for
metadata and Allbert Home filesystem for YAML bodies; both have
`tiles_for_thread/2` / `surfaces_for_thread/2` for read,
`add_tile/1` / `open/1` for create, `update_tile/2` / `dismiss/3`
for mutation. Soft-delete via `deleted_at`. Bounded to
`workspace.canvas.max_tiles_per_thread` /
`workspace.ephemeral.max_active_per_thread`.

**Canvas cap-exceeded policy (FIFO eviction with pinning).** When
`add_tile/1` would push the per-thread tile count above
`workspace.canvas.max_tiles_per_thread`, Canvas evicts the oldest
non-pinned tile (FIFO by `inserted_at`). Eviction is a soft-delete:
`deleted_at` is set in SQLite and the body YAML moves to a sibling
`.deleted.<ts>.yml` path under the same canvas directory. Pinned
tiles (`pinned = true`) are exempt from FIFO eviction. If every tile
in the thread is pinned and the cap is exceeded, `add_tile/1`
returns `{:error, :canvas_cap_exceeded}` and the emitter receives
the error through Fragment validation. Each eviction emits an
`allbert.workspace.tile.removed` signal with `removed_reason:
:cap_evicted` plus a `:badge_strip` Fragment for the canvas header
("N older tile(s) archived — view via `mix allbert.workspace canvas
list --include-deleted`"). Eviction never destroys body content;
operators recover via `Canvas.restore_tile/2` or
`mix allbert.workspace canvas restore <tile-id>` (which clears
`deleted_at`, moves the YAML back, and increments position to the
end). Hard deletion remains a follow-up command
(`mix allbert.workspace canvas purge --before <date>`).

`pin_tile/2` and `unpin_tile/2` toggle the `pinned` boolean,
emit `allbert.workspace.tile.updated` with `changed_fields:
[:pinned]`, and bypass the `:workspace_canvas_write` permission
check ONLY when the actor matches the tile's `user_id`.

`Fragment` owns the emission pipeline:

```elixir
defmodule AllbertAssist.Workspace.Fragment do
  alias AllbertAssist.Workspace.Fragment.Envelope

  @spec emit(envelope :: Envelope.t()) :: :ok | {:error, atom()}
  def emit(envelope) do
    with :ok <- validate_envelope_shape(envelope),
         :ok <- verify_signature(envelope),
         :ok <- validate_surface(envelope.surface),
         :ok <- validate_emitter(envelope.emitter_id),
         :ok <- check_rate_limit(envelope.emitter_id, envelope.user_id),
         :ok <- check_size(envelope) do
      Jido.Signal.Bus.publish(AllbertAssist.SignalBus,
        "allbert.workspace.fragment.emitted", envelope)
    end
  end
end
```

`FragmentEnvelope` carries the bounded provenance wrapper (surface,
emitter_id, user_id, thread_id, scope, tile_position, kind,
emitted_at, signature). HMAC-SHA256 signature keyed by
`workspace.fragment.signing_secret` (auto-generated at first boot,
persisted to `<ALLBERT_HOME>/workspace/secrets/`, rotatable via
`mix allbert.workspace rotate-signing-secret`).

### `AllbertAssist.Workspace.AGUI.Bridge` (internal, test-only)

Translates a curated subset of Allbert SignalBus events to AG-UI
event-shape JSON for testing the semantic mapping early.
**NOT exposed over HTTP / WebSocket / SSE.** Documented mapping:

| Allbert signal | AG-UI event |
|---|---|
| `allbert.runtime.turn.started` | `LIFECYCLE_START` |
| `allbert.runtime.turn.completed` | `LIFECYCLE_END` |
| `allbert.confirmation.requested` | `INTERRUPT` |
| `allbert.confirmation.approved` | `INTERRUPT_RESPONSE` (approve) |
| `allbert.confirmation.denied` | `INTERRUPT_RESPONSE` (reject) |
| `allbert.objective.observed` | `STATE_DELTA` |
| `allbert.objective.completed` | `STATE_SNAPSHOT` |
| `allbert.action.requested` | `TOOL_CALL_START` |
| `allbert.action.completed` | `TOOL_CALL_END` |
| `allbert.action.failed` | `TOOL_CALL_ERROR` |

Unmapped signals return `{:error, :no_mapping}` (silent drop —
the bridge is a partial translation by design).

### `AllbertAssist.Workspace.Offline`

Server-side offline reconciliation boundary. The server does not
interpret Yjs internals in v0.26 and does not add a Rust NIF. The
browser hook owns the Yjs document and IndexedDB persistence; Elixir
validates provenance, bounds update payloads, records revision rows,
stores the latest human-readable snapshot, and broadcasts the
reconciled state. API:

```elixir
@spec record_client_update(attrs :: map()) ::
        {:ok, revision :: map()}
        | {:error, term()}

@spec latest_snapshot(tile_id :: String.t(), user_id :: String.t()) ::
        {:ok, String.t()} | {:error, term()}

@spec pending_conflict_summary(tile_id :: String.t(), user_id :: String.t()) ::
        {:ok, map()} | {:error, term()}
```

The accepted browser contract for M18-M19:

- A `WorkspaceTileEditor` hook creates one `Y.Doc` per editable text
  or markdown tile.
- The hook uses a shared Yjs text/map type for the tile body,
  persists local state through `y-indexeddb`, and pushes bounded
  update payloads to LiveView with `pushEvent`.
- The hook uses `phx-update="ignore"` only for the editor-owned DOM
  subtree so LiveView does not fight the client editor.
- Reconnect sync sends base revision, state vector, pending Yjs
  updates, and a text/markdown snapshot. Server-side code stores the
  opaque update blob plus snapshot; it does not compile or execute
  client-provided code.

- **No drag-drop tile reordering.** Resize-bar only between regions.

- **No canvas snapshot / undo / time-travel.** Reserved in ADR 0023.
- **No public AG-UI HTTP endpoint.** Internal bridge only.
- **No A2UI / MCP Apps interop.** Post-v0.31.
- **No multi-user collaborative cursors.** Post-v0.31.
- **No plugin-contributed workspace regions.** Post-v0.31.
- **No offline editing of non-text tile kinds.** Confirmation cards
  + approval inspectors + trace viewers are read-only offline.
- **No real StockSage card rendering.** v0.27 implements.
- **No new app contract or SurfaceProvider behaviour change.** v0.18
  contract stays; only catalog expands.
- **No new permission framework.** Security Central unchanged.
- **No model-generated HTML/JS/Elixir.** Declarative + catalog-bound
  only.
