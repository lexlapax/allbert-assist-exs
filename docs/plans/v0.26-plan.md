# Allbert v0.26 StockSage Polish, Outcomes, And Trends

Formerly tracked as M-D3b.

## Purpose

v0.26 brings StockSage's Elixir implementation to parity with Python
StockSage 0.0.2 user-facing behavior: outcomes, reflections, memory sync,
trends, reruns, and polished responsive UI.

## Expected Inputs

- v0.23 StockSage LiveViews exist.
- v0.18 StockSage outcome and memory-entry tables exist.
- v0.20 bridge remains available, or v0.22 native parity has passed.
- v0.19 memory review semantics distinguish conversation history from durable
  markdown memory.

## Expected Direction

- Add outcome resolver worker for post-holding-period returns using the v0.18
  `stocksage_outcomes` table.
- Generate LLM reflection for resolved outcomes through registered actions.
- Sync resolved lessons into the v0.18 StockSage memory records and Allbert
  memory namespaces through explicit, traceable actions.
- Add alpha-aware accuracy, rating calibration, and leaderboard trends.
- Add analysis rerun from LiveView.
- Add mobile-responsive layout, error states, and empty states.

## Non-Goals

- No canvas tile work.
- No hosted deployment requirement.
- No automatic promotion of every analysis into markdown memory.

## Test Plan

- Outcome resolver updates records idempotently.
- Trends calculations match fixture expectations.
- Memory sync is explicit and traceable.
- LiveView rerun, empty, error, and mobile layouts pass focused tests/manual
  review.

## Definition Of Done

v0.26 is done when all Python StockSage 0.0.2 user-facing features are
replicated in Elixir without requiring a Python runtime if v0.22 native parity
has passed.
