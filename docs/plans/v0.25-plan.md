# Allbert v0.25 StockSage LiveViews

Formerly tracked as M-D3a, previously planned as v0.23, then v0.24. Redesigned to build on
the v0.18 app/surface contract and Surface DSL from day one.

## Purpose

v0.25 adds StockSage's Phoenix LiveView surfaces using the v0.18 app/surface
contract and `AllbertAssist.Surface` DSL as the foundation. It proves that a
plugin-contributed app can expose web surfaces through
`AllbertAssist.App.SurfaceProvider` from day one — there is no static route
mounting that later migrates to a surface contract.

This milestone deliberately follows both analysis engines (v0.22 bridge,
v0.23 native agents) and the v0.18 app/surface contract so the web surface is
built on the right contracts from the start and supports both execution modes.

## Expected Inputs

- v0.14 session scratchpad provides `active_app`.
- v0.15 minimal app registration and v0.18 app/surface contract exist.
  `AllbertAssist.App.SurfaceProvider` behaviour and `AllbertAssist.Surface`
  DSL with catalog validation are available. `StockSage.App` already implements
  the v0.18 app/surface contract with placeholder StockSage surface metadata.
- v0.20 StockSage contexts, queue records, and imported domain records exist.
- v0.22 Python bridge provides real analyses.
- v0.23 Native Jido agents: both bridge and native analysis engines are
  available. The web surface represents both from day one.

## Expected Direction

- Add `StockSageLive.WorkspaceLive`, `AnalysisLive`, `QueueLive`, and
  `TrendsLive` in `apps/stocksage_web`.
- Extend the existing `StockSage.App` `SurfaceProvider` implementation with
  concrete `/stocksage/...` routes and component catalog entries. Declare
  StockSage navigation surfaces through `AllbertAssist.Surface` DSL. This is
  how v0.24's workspace shell discovers StockSage surfaces via
  `AllbertAssist.App.Registry.registered_apps/0` without a private router
  inventory.
- Mount `/stocksage/...` routes in the Allbert web router for LiveView
  rendering. Routes live in the router; navigation metadata comes from
  `SurfaceProvider`. These are complementary, not duplicating.
- Declare StockSage component catalog entries (`:analysis_card`,
  `:queue_entry`, `:trend_summary`) through the Surface DSL. These types are
  declared but not canvas-registered yet — that is v0.28. Declaring them here
  means v0.28 has no format migration.
- Update `StockSage.Actions.RunAnalysis` results to include an
  `AllbertAssist.Surface.Node` of type `:analysis_card`. The node is validated
  against the catalog at action completion. The workspace shell (v0.24) and
  canvas (v0.28) consume this existing format without changes.
- Stream analysis progress through Phoenix.PubSub and LiveView `stream/3`.
- Set `active_app: :stocksage` in scratchpad when navigating under
  `/stocksage/`.
- Keep LiveViews thin: domain behavior stays behind StockSage contexts,
  registered actions, and runtime boundaries.

## Non-Goals

- No canvas tile registration; that is v0.28.
- No AG-UI protocol emission; Surface nodes are Allbert-native validated
  structures. The `to_a2ui/1` stub exists in v0.18 but is not wired to
  LiveView output.
- No `canvas_ops`; that is v0.28.
- No arbitrary model-generated component shapes.
- No PostgreSQL or Oban dependency.

## Test Plan

- `StockSage.App` implements `AllbertAssist.App.SurfaceProvider`;
  `mix allbert.validate_app StockSage.App` passes.
- `AllbertAssist.App.Registry.registered_apps/0` includes StockSage with
  declared surface paths. Workspace navigation will be available from v0.24
  without further changes.
- StockSage LiveViews load from `/stocksage/...` routes.
- `active_app: :stocksage` is set in scratchpad on navigation to StockSage
  routes.
- `RunAnalysis` action results carry a validated `:analysis_card`
  `AllbertAssist.Surface.Node`.
- A Surface.Node carrying an undeclared component type is rejected by catalog
  validation.
- Live progress updates stream without polling or page reload.
- Both bridge and native analysis results are representable through the same
  LiveViews.
- Full gates: compile warnings-as-errors, format check, Credo strict,
  Dialyzer, precommit.

## Definition Of Done

v0.25 is done when StockSage has Phoenix LiveViews accessible through declared
`SurfaceProvider` surfaces, workspace navigation can discover StockSage from
`AllbertAssist.App.Registry`, `RunAnalysis` results carry validated
`AllbertAssist.Surface.Node` structures, the StockSage component catalog is
declared as the foundation for v0.28 canvas integration, and both analysis
engines are representable in the web surface.
