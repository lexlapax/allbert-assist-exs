# Allbert v0.19 Cross-Surface Intent Enrichment

## Purpose

v0.19 finishes the broader intent-engine ambition after Allbert has real
security decisions, skills, confirmations, execution adapters, jobs, channels,
memory and trace signals, and an app/surface contract with registered surface
metadata.

v0.11 introduced the execution-aware intent decision contract early so risky
capabilities could stay explainable. v0.19 enriches that contract with
cross-surface routing, candidate ranking, optional model assistance, and evals
grounded in actual runtime behavior.

This milestone was formerly tracked as v0.15 before the post-v0.10 roadmap
was unified into one v0.xx sequence, then tracked as v0.21, then v0.22, as
the roadmap was resequenced.

## Expected Inputs

- v0.05 Security Central decisions exist.
- v0.11 intent decisions exist and cover risky execution flows.
- v0.13 scheduled jobs provide background execution context.
- v0.16 additional channels provide channel metadata and handoff behavior.
- v0.17 plugin registry provides plugin provenance for channel, skill, action,
  and app contributions.
- v0.14 session scratchpad provides bounded session context, including
  `active_app` when the operator is working inside an app surface.
- v0.15 minimal app registration and v0.18 app/surface contract provide
  app-scoped action, skill, and surface metadata. `AllbertAssist.App.Registry`
  contains registered app navigation surfaces from `SurfaceProvider`.

## Expected Direction

- Move beyond one-off route predicates into a hybrid deterministic and
  model-assisted intent engine that remains testable.
- Use settings, skill registry, action-backed skill contracts, Security Central
  decisions, confirmation history, execution traces, jobs, channels, and
  existing memory metadata as routing inputs.
- Leave reviewed-memory retrieval as an additive v0.21 input. v0.19 should
  expose an intent contract that v0.21 can plug reviewed memory into without
  creating a forward dependency.
- Use plugin provenance as explainability/ranking metadata only. A plugin tag
  can help explain why an action, skill, or channel exists, but it never
  authorizes execution or bypasses trust policy.
- Use `active_app` from the session scratchpad to scope candidate action
  ranking. Actions registered by the active app are prioritized only when that
  session context is present. Cross-app routing requires explicit `active_app`
  evidence, and app-specific skill paths registered through `AllbertAssist.App`
  are included in candidate skill ranking.
- Use registered app surface metadata from `AllbertAssist.App.Registry` to
  include surface navigation as a routing target when `active_app` evidence is
  present. A routing outcome of "open StockSage workspace" is valid only when
  StockSage session context exists.
- Rank candidate skills/actions/jobs/channels while keeping untrusted skills and
  unregistered actions out of executable decisions.
- Optionally ask a model for structured classification when deterministic
  confidence is low.
- Validate model output against known skills, actions, permissions, execution
  modes, channels, and allowed statuses.
- Add eval fixtures for activation, non-activation, safe action, confirmation,
  denial, shell/script/package/network/import, job, channel, memory, and
  refusal cases.

## Non-Goals

- No new shell, script, package, network, or import powers.
- No unsupervised model-selected side effects.
- No opaque memory mutation.
- No channel-specific intent forks.

## Definition Of Done

v0.19 is done when:

- Runtime turns pass through the enriched intent engine.
- Existing v0.11/v0.15 intent behavior is preserved.
- Skill/action/job/channel selection is registry-aware and security-aware.
- Intent traces explain selected and rejected candidates.
- Optional model-assisted classification is validated, bounded, and testable.
- Confirmation, jobs, channels, existing memory metadata, and registered app
  surfaces have one shared intent contract to consume; v0.21 memory review can
  later add reviewed memory context through that same contract.
