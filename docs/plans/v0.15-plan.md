# Allbert v0.15 Minimal App Registration Contract

Formerly tracked as M-AppContract-Lite.

## Purpose

v0.15 gives StockSage and future workspace apps a small public surface to
register actions, skills, and navigation with Allbert without reaching through
private internals. This is the minimal app contract; the full app/surface DSL
arrives in v0.24.

## Expected Inputs

- v0.12 local string `user_id` exists in runtime.
- v0.14 session scratchpad can carry `active_app`.
- `AllbertAssist.Actions.Registry` and `AllbertAssist.Skills.Registry` exist.
- ADR 0015 is accepted before implementation.

## Expected Direction

- Add `AllbertAssist.App` lite behaviour with required callbacks:
  `app_id/0`, `display_name/0`, `version/0`, `validate/1`, `child_spec/1`,
  `actions/0`, and `skill_paths/0`.
- Add optional `surfaces/0` for shell navigation entries.
- Add `AllbertAssist.App.Registry` with `register/2`, `lookup/1`,
  `registered_apps/0`, and `registered_surfaces/0`.
- Add `AllbertAssist.App.Supervisor` as a `DynamicSupervisor`.
- Allow `AllbertAssist.Actions.Registry` entries to carry optional `app_id`.
- Register app skill paths through the existing skill registry.
- Keep routes statically configured; navigation entries are display data, not
  dynamic route injection.

## Non-Goals

- No `AllbertAssist.Surface` DSL.
- No `AllbertAssist.App.SurfaceProvider`.
- No AG-UI or A2UI dependency.
- No generator.
- No permission grants from app registration.

## Test Plan

- A minimal test app registers one action, one skill path, and one navigation
  surface.
- Duplicate app ids fail cleanly.
- Failed `validate/1` prevents registration without crashing the supervision
  tree.
- Core actions and skills continue to resolve unchanged.
- Registered app actions are tagged with `app_id`.

## Definition Of Done

v0.15 is done when a minimal test app can register and appear in app, skill,
action, and surface lookups while existing Allbert runtime behavior is
unchanged.
