# Allbert v0.15 Cross-Surface Intent Enrichment

## Purpose

v0.15 finishes the broader intent-engine ambition after Allbert has real
security decisions, skills, confirmations, execution adapters, jobs, channels,
and memory review signals.

v0.11 introduced the execution-aware intent decision contract early so risky
capabilities could stay explainable. v0.15 enriches that contract with
cross-surface routing, candidate ranking, optional model assistance, and evals
grounded in actual runtime behavior.

## Expected Inputs

- v0.05 Security Central decisions exist.
- v0.11 intent decisions exist and cover risky execution flows.
- v0.12 scheduled jobs provide background execution context.
- v0.13 additional channels provide channel metadata and handoff behavior.
- v0.14 memory review and retrieval provide reviewed memory context.

## Expected Direction

- Move beyond one-off route predicates into a hybrid deterministic and
  model-assisted intent engine that remains testable.
- Use settings, skill registry, action-backed skill contracts, Security Central
  decisions, confirmation history, execution traces, jobs, channels, and memory
  review signals as routing inputs.
- Rank candidate skills/actions/jobs/channels while keeping untrusted skills and
  unregistered actions out of executable decisions.
- Optionally ask a model for structured classification when deterministic
  confidence is low.
- Validate model output against known skills, actions, permissions, execution
  modes, channels, and allowed statuses.
- Add eval fixtures for activation, non-activation, safe action, confirmation,
  denial, shell/script/package/network/import, job, channel, memory, and
  refusal cases.

## Non-Goals

- No new shell, script, package, network, or import powers.
- No unsupervised model-selected side effects.
- No opaque memory mutation.
- No channel-specific intent forks.

## Definition Of Done

v0.15 is done when:

- Runtime turns pass through the enriched intent engine.
- Existing v0.14 behavior is preserved.
- Skill/action/job/channel selection is registry-aware and security-aware.
- Intent traces explain selected and rejected candidates.
- Optional model-assisted classification is validated, bounded, and testable.
- Confirmation, jobs, channels, and memory review have one shared intent
  contract to consume.
