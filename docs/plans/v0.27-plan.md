# Allbert v0.27 StockSage Polish, Outcomes, And Trends

Formerly tracked as M-D3b.

## Purpose

v0.27 brings StockSage's Elixir implementation to parity with Python
StockSage 0.0.2 user-facing behavior: outcomes, reflections, memory sync,
trends, reruns, and polished responsive UI. It also completes the final deferred
layer of the v0.18 app contract by adding memory namespace registration.

## Expected Inputs

- v0.25 StockSage LiveViews exist on `AllbertAssist.App.SurfaceProvider`.
- v0.20 StockSage outcome and memory-entry tables exist.
- v0.22 bridge remains available, or v0.23 native parity has passed.
- v0.21 memory review semantics distinguish conversation history from durable
  markdown memory.
- v0.18 app/surface contract provides all app contract layers except memory
  namespace registration, which this milestone adds.

## Expected Direction

- Add memory namespace registration to `AllbertAssist.App.Registry` as the
  final deferred layer of the v0.18 app contract. Update `StockSage.App`
  to declare its memory namespace. This is the prerequisite for syncing
  resolved lessons into Allbert memory namespaces.
- Add outcome resolver worker for post-holding-period returns using the v0.20
  `stocksage_outcomes` table.
- Generate LLM reflection for resolved outcomes through registered actions.
- Sync resolved lessons into the v0.20 StockSage memory records and Allbert
  memory namespaces through explicit, traceable actions.
- Add alpha-aware accuracy, rating calibration, and leaderboard trends.
- Add analysis rerun from LiveView.
- Add mobile-responsive layout, error states, and empty states.

## Non-Goals

- No canvas tile work.
- No hosted deployment requirement.
- No automatic promotion of every analysis into markdown memory.

## Test Plan

- Memory namespace registration in `AllbertAssist.App.Registry` works for
  StockSage; `mix allbert.validate_app StockSage.App` passes with the memory
  namespace callback implemented.
- Outcome resolver updates records idempotently.
- Trends calculations match fixture expectations.
- Memory sync is explicit and traceable.
- LiveView rerun, empty, error, and mobile layouts pass focused tests/manual
  review.

## Definition Of Done

v0.27 is done when all Python StockSage 0.0.2 user-facing features are
replicated in Elixir without requiring a Python runtime if v0.23 native parity
has passed, and the v0.18 app contract is fully complete including memory
namespace registration.
