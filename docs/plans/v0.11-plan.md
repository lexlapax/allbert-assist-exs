# Allbert v0.11 Execution-Aware Intent Contract

## Purpose

v0.11 introduces the first structured intent decision contract over real risky
capabilities.

Allbert needs this before scheduled jobs or new channels consume shell, script,
package, network, and online import capabilities. The contract makes selection,
confirmation, denial, and refusal explainable without adding new execution
powers.

## Expected Inputs

- v0.05 Security Central decision shape exists.
- v0.06 action-backed skills and action runner exist.
- v0.07 confirmation workflow exists.
- v0.08 shell execution exists behind sandbox policy.
- v0.09 skill script execution exists behind trusted resource policy.
- v0.10 external service, package-install, and online import adapters exist.

## Intent Decision Contract

Introduce a structured intent decision, tentatively
`AllbertAssist.Intent.Decision`, with fields equivalent to:

- `:intent`
- `:confidence`
- `:reason`
- `:risk_summary`
- `:selected_skill`
- `:candidate_skills`
- `:selected_action`
- `:candidate_actions`
- `:permission`
- `:confirmation`
- `:execution_mode`
- `:foreground_or_background`
- `:alternatives`
- `:diagnostics`
- `:trace_metadata`

The decision object is plain data. It does not execute anything. Runtime still
executes through registered actions, the action lifecycle runner, Security
Central, the permission gate compatibility layer, and confirmation state.

## Non-Goals

- No new execution powers.
- No model-selected side effects without validation.
- No broad cross-channel routing; v0.15 owns that.
- No memory retrieval or embeddings work.
- No channel-specific intent fork.

## Definition Of Done

v0.11 is done when:

- Runtime turns can include an intent decision object.
- Risky capability prompts produce inspectable selected/candidate decisions
  before execution or confirmation.
- Invalid model or route output cannot invent skill names, action names,
  permissions, or execution modes.
- Intent traces explain selected and rejected candidates.
