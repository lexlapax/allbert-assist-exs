# Allbert v0.28 Allbert App Generator

## Purpose

v0.28 adds a generator only after StockSage proves the full app contract and
canvas path. The generator should encode the boring, proven shape of an
Allbert app rather than inventing abstractions early.

## Expected Inputs

- v0.24 full app contract exists.
- v0.25 proves StockSage as a full app.
- v0.26 workspace/canvas substrate exists.
- v0.27 proves the app/canvas path with StockSage canvas components.

## Expected Direction

- Add `mix allbert.gen.app MyApp`.
- Generate the boring local app shape that v0.15, v0.24, v0.26, and v0.27
  proved: app module, app supervision wiring, sample Jido action, sample
  `SKILL.md`, sample surface provider or surface node, sample Ecto domain
  stub, and validation documentation.
- Ensure `mix allbert.validate_app MyApp` passes immediately.
- Generated code is inert by default. It does not trust or enable generated
  skills, grant action permissions, register external publishing, or perform
  side effects without explicit operator action through existing Allbert
  boundaries.
- Consider optional `mix allbert.publish_skills` only after local validation,
  trust, and explicit operator review are boring and safe.

## Non-Goals

- No generator before the contract is proven.
- No autonomous app or skill creation from traces, prompts, or repeated tasks.
- No runtime dynamic module loading from arbitrary app or skill folders.
- No generated execution authority, automatic trust, automatic skill
  enablement, or automatic permission grants.
- No marketplace publish without explicit operator action and review.
- No AG-UI, A2UI, MCP Apps, or external UI protocol bridge.

## Test Plan

- Generated app compiles and validates.
- Generated skill is inert until explicitly enabled/trusted.
- Generated action goes through the registry, action runner, and Security
  Central.
- Generated surface data validates through `AllbertAssist.Surface` and does
  not invent component ids or action bindings.
- Generator output includes clear validation docs for the created app.

## Definition Of Done

v0.28 is done when a new local Allbert app can be scaffolded, validated, and
inspected without granting unsafe capability, publishing externally, or loading
untrusted runtime code.
