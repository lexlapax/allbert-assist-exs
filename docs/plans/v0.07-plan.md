# Allbert v0.07 Skill Script Runner

## Purpose

v0.07 lets trusted Agent Skills run selected bundled scripts, but only through
Allbert's action, confirmation, sandbox, and trace boundaries.

v0.03 already inventories `scripts/` as inert resources. v0.06 provides a
local execution sandbox. v0.07 connects those pieces with a stricter rule:
scripts are executable only when they are part of the selected skill's parsed
resource inventory and a registered action chooses to run them.

## Expected Inputs

- v0.03 resource inventory includes bundled skill scripts.
- v0.04 validates trusted skill capability contracts against registered
  actions.
- v0.05 confirmation workflow can approve or deny risky actions.
- v0.06 local execution sandbox can execute bounded commands with traces.

## Expected Direction

- Add `run_skill_script` as a registered Jido action with a schema for selected
  skill, script resource path, args, working directory policy, timeout, and
  expected output mode.
- Resolve script paths only from the selected skill's parsed resource
  inventory; reject absolute paths, path traversal, hidden files, missing
  resources, and non-script resources.
- Require the skill to be trusted, enabled, selected for the current action
  context, and allowed by its validated capability contract.
- Run scripts through the v0.06 sandbox and v0.05 confirmation workflow.
- Trace script source path, resource digest, args summary, permission,
  confirmation, stdout/stderr summary, exit status, timeout, and failure reason.

## Non-Goals

- No arbitrary script execution outside a trusted skill resource inventory.
- No runtime loading of Elixir modules from skill folders.
- No package installs or dependency bootstrap; v0.08 owns install policy.
- No external service adapter from script metadata.
- No persistent background script processes.
- No auto-enabling imported or untrusted skills.

## Definition Of Ready

v0.07 is ready to implement when:

- v0.06 sandbox behavior is tested and traceable.
- v0.05 confirmation can resume actions with selected skill context.
- v0.04 contract validation can represent script execution policy without
  treating metadata as permission.

## Definition Of Done

v0.07 is done when:

- `run_skill_script` exists only as a registered action.
- Trusted inventoried scripts can run after confirmation through the sandbox.
- Untrusted, disabled, non-selected, non-inventoried, or path-escaping scripts
  are refused before execution.
- Traces include skill, script, resource digest, permission, confirmation, and
  execution outcome.
- Skill activation alone still never executes scripts.
