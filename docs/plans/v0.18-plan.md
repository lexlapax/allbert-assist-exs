# Allbert v0.18 Full App Contract And Surface DSL

Formerly tracked as M-AppContract-Full, previously planned as v0.25, then v0.21.

## Purpose

v0.18 matures the minimal app registration contract into the full app and
surface contract, except for memory namespace registration, and adds the
`AllbertAssist.Surface` DSL that all app web surfaces must use. It also
upgrades `AllbertAssist.App.CoreApp` — the built-in `:allbert` app — into the
first real `SurfaceProvider`, formally declaring the `/agent` conversation
route as a built-in surface. This makes the conversational chat surface the
first proven implementation of the app/surface contract before any
plugin-contributed app (StockSage v0.20) builds on it.

Channels that deliver requests without another `active_app` context default to
`active_app: :allbert`, so every runtime turn has a declared home app. v0.24
later upgrades `CoreApp`'s surface into the full workspace shell; it is
`CoreApp`'s surface implementation, not a separate shell floating outside the
app contract.

Memory namespace registration is deferred to v0.27, where StockSage polish
first consumes it. All other contract layers are complete in this milestone.

## Expected Inputs

- v0.15 minimal app contract exists.
- v0.17 plugin contract exists.
- ADR 0015 is accepted and updated with any implementation decisions.

## Expected Direction

- Extend `AllbertAssist.App` to include identity/OTP, agents/actions/signals,
  skills, UI surfaces, and data/settings layers. Memory namespaces are deferred
  to v0.27.
- Extend `AllbertAssist.App.Registry` with signal subscriptions and settings
  validation. Memory namespace registration is deferred to v0.27.
- Keep plugin registration as the package/discovery layer. The app contract
  should expand app capabilities without turning `AllbertAssist.App.Registry`
  into the general plugin registry.
- Add `AllbertAssist.App.SurfaceProvider`.
- Add `AllbertAssist.Surface` with catalog validation over known node shapes.
- Add optional `AllbertAssist.Surface.Encoder.to_a2ui/1`; do not use it in the
  Phoenix LiveView path. This encoder is the designated AG-UI adaptation stub.
  Its type signature should document the intended translation from
  `AllbertAssist.Surface.Node` to AG-UI `STATE_SNAPSHOT`-style events. The
  implementation may return `:not_implemented` in this milestone; a concrete
  AG-UI adapter is post-v0.29 work.
- Upgrade `AllbertAssist.App.CoreApp` to implement `SurfaceProvider`. Declare
  the `/agent` conversation route as `CoreApp`'s built-in surface with a
  `:chat` component catalog entry for the conversation timeline. This is the
  first `SurfaceProvider` implementation and proves the contract before
  StockSage (v0.20) uses it.
- Set `active_app: :allbert` as the default for runtime turns that arrive
  without another app context — specifically channel-delivered requests (v0.16
  Telegram/email) when no `active_app` has been set in scratchpad. This gives
  every turn a declared home app and makes `CoreApp` the channel fallback.
- Add `mix allbert.validate_app MyApp`.
- Design the app/surface contract so v0.20 StockSage can implement
  `AllbertAssist.App.SurfaceProvider` as the second consumer from day one.
  There is no stepping-stone v0.15 → v0.18 contract migration needed in v0.20.
- Add `docs/how-to-create-an-allbert-app.md`.

## Non-Goals

- No app generator; that is v0.29.
- No AG-UI/A2UI package dependency for the local web path.
- No canvas implementation; that is v0.24.
- No permission grants from app registration.
- No memory namespace registration in `AllbertAssist.App.Registry`; that is
  v0.27.
- No AG-UI protocol emission in this milestone; the Surface DSL is designed to
  support a future AG-UI adapter without encoding protocol-specific wire format.

## Test Plan

- `mix allbert.validate_app AllbertAssist.App.CoreApp` passes. `CoreApp`
  declares the `/agent` surface route through `SurfaceProvider`.
- `AllbertAssist.App.Registry.registered_apps/0` includes `CoreApp` with a
  declared surface path; workspace navigation (v0.24) requires no further
  change to discover it.
- Channel-delivered requests with no prior `active_app` resolve to
  `active_app: :allbert` in the runtime turn context.
- `mix allbert.validate_app` passes for a conforming test app; invalid
  callbacks or missing required callbacks are rejected.
- Plugin inspection shows a test app's contribution without granting authority.
- Invalid callbacks or invalid surfaces fail validation.
- Surface DSL rejects model-invented component names and action bindings.
- `AllbertAssist.Surface.Encoder.to_a2ui/1` type signature is present and
  documented; the stub does not emit AG-UI wire-format events.
- App registration still does not bypass Security Central or action runner.

## Definition Of Done

v0.18 is done when the app/surface contract is implemented through all layers
except memory namespace registration, `AllbertAssist.App.CoreApp` implements
`SurfaceProvider` and declares the `/agent` conversation surface as the first
built-in chat surface, channel-delivered requests default to
`active_app: :allbert` when no other app context is active, surface validation
exists through `AllbertAssist.Surface` DSL, `mix allbert.validate_app` passes
for a conforming app, the `to_a2ui/1` encoder stub is typed and documented,
ADR 0015 is updated, and v0.20 StockSage can implement the same app/surface
contract from day one without any stepping-stone migration from the v0.15
minimal contract. Memory namespace registration remains the final deferred
app-contract layer for v0.27.
