# Allbert v0.13 Cross-Surface Intent Enrichment

## Purpose

v0.13 finishes the broader intent-engine ambition after Allbert has real
skills, confirmations, execution adapters, jobs, channels, and memory review
signals.

v0.09 introduced the execution-aware intent decision contract early so risky
capabilities could stay explainable. v0.13 enriches that contract with
cross-surface routing, candidate ranking, optional model assistance, and evals
grounded in actual runtime behavior.

## Expected Inputs

- v0.09 intent decisions exist and cover risky execution flows.
- v0.10 scheduled jobs provide background execution context.
- v0.11 additional channels provide channel metadata and handoff behavior.
- v0.12 memory review and retrieval provide reviewed memory context.
- Permission decisions remain centralized in
  `AllbertAssist.Security.PermissionGate`.

## Non-Goals

- No new shell, script, package, network, or import powers.
- No unsupervised model-selected side effects.
- No opaque memory mutation.
- No channel-specific intent forks.

## Expected Direction

- Move beyond one-off route predicates into a hybrid deterministic and
  model-assisted intent engine that remains testable.
- Use settings, skill registry, action-backed skill contracts, confirmation
  history, execution traces, jobs, channels, and memory review signals as
  routing inputs.
- Rank candidate skills/actions/jobs/channels while keeping untrusted skills and
  unregistered actions out of executable decisions.
- Optionally ask a model for structured classification when deterministic
  confidence is low.
- Validate model output against known skills, actions, permissions, execution
  modes, channels, and allowed statuses.
- Add eval fixtures for activation, non-activation, safe action, confirmation,
  denial, shell/script/package/network/import, job, channel, memory, and
  refusal cases.

## Definition Of Ready

v0.13 is ready to implement when:

- v0.09 decisions are stable in runtime traces.
- Jobs, at least one additional channel, and memory review produce real
  decision pressure.
- Deterministic fixtures exist for current intent behavior.

## Definition Of Done

v0.13 is done when:

- Runtime turns pass through the enriched intent engine.
- Existing v0.12 behavior is preserved.
- Skill/action/job/channel selection is registry-aware and permission-aware.
- Intent traces explain selected and rejected candidates.
- Optional model-assisted classification is validated, bounded, and testable.
- Confirmation, jobs, channels, and memory review have one shared intent
  contract to consume.
